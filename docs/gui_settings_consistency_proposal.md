# GUI 設定の見せ方整理 — 提案書

**作成日**: 2026-04-26
**ステータス**: Draft (未承認、議論中)
**関連 PR**: #135 (auto_blur), #136 (Text+ 自動変換), #137 (タイトル画像修正)

## 1. 背景と動機

TextffCut の GUI には複数レイヤの設定が混在している:

- 動画ごとに 1 度設定するもの (auto_blur 適用、preset_dir)
- クリップ生成毎に都度変えるもの (候補数、速度、ズーム、メディア素材)
- 出力経路毎に変えるもの (DaVinci 送信時の Text+ 変換、subtitle 無効化)

機能追加 (auto_blur / Text+ 自動変換) に伴って設定が画面間で散らばり、**ユーザーが「どこで何を設定するか」がわからなくなりつつある**。本提案ではレイヤモデルを定義し、各画面の改善案を提示する。

## 2. 現状の問題点

### 2.1 auto_blur (塗りつぶし) の設定が画面間で不揃い

| 画面 | チェックボックス | 役割 |
|---|---|---|
| 文字起こし画面 | ☑ 🔒 動画内テキスト自動塗りつぶし | 塗りつぶし版動画を**生成**するか |
| FCPXML エクスポート画面 | ☑ 🔒 塗りつぶし版動画をソースとして利用 | 生成済み cache を**使用**するか (cache 存在時のみ) |
| **AI自動切り抜きセクション** | **なし** ⚠ | 内部デフォ True で動作するが UI から制御不能 |

ユーザーが AI自動切り抜きを使う場合、塗りつぶし版を「使うか」を**チェックボックスから切り替える手段がない**。

### 2.2 Text+ 自動変換が「DaVinciへ送信」ボタン直前のみ

`📺 DaVinciへ送信` ボタンの直前にチェックボックスを置く設計 (PR #136)。1 出力経路に紐づくので合理的だが、他の出力経路 (`textffcut clip` → 別途送信、CLI 直接呼び出し) では設定値が同期しない。

### 2.3 設定のグループ化が一貫していない

| 設定カテゴリ | 集約状況 |
|---|---|
| メディア素材 (frame/BGM/SE/title) | ✅ 「🎨メディア素材」セクションで集約 |
| 動画処理パラメータ (speed/zoom/anchor) | △ AI自動切り抜きセクション内、メディア素材より上 |
| auto_blur (塗りつぶし) | ❌ 文字起こし画面と FCPXML エクスポート画面に分散、AI 切り抜きには無し |
| Text+ 連携 | △ 字幕エディタの送信ボタン直前のみ |

### 2.4 「いつ変える設定か」のルールが暗黙

ユーザーが設定を変える頻度に応じた階層化がないため、「動画ごとに 1 度設定すれば済むもの」と「クリップ毎に微調整するもの」が同じ画面に並んでいる。

## 3. 改善方針 — 設定の 3 階層モデル

設定を頻度・スコープで 3 階層に分類する:

### 3.1 階層定義

| 階層 | スコープ | 設定例 | 推奨 UI 配置 |
|---|---|---|---|
| **動画レベル** | 動画ごと、cache に紐づく | auto_blur 生成、preset_dir | サイドバー or 文字起こし画面 (1 度設定して以降は表示しない) |
| **クリップレベル** | クリップ生成毎 | 候補数、min/max duration、速度、ズーム、アンカー、メディア素材、**塗りつぶし版利用** | AI自動切り抜きセクション内 |
| **出力レベル** | 出力経路 (送信/エクスポート) 毎 | Text+ 変換、subtitle 無効化、送信先ビン名、フォーマット選択 | 各出力ボタン直前 (現状の Text+ チェックと同じ流儀) |

### 3.2 設計原則

1. **同じスコープの設定は同じセクションに集める**
2. **動画レベル設定は cache が一致する限り変えない** (= 一度設定したら以降の操作で再表示しない、確認のみ)
3. **クリップレベル設定は前回値を session_state に永続化** (ユーザーが「いつもの設定」を再利用しやすく)
4. **出力レベル設定は出力ボタン直前に置く** (押す直前まで判断を遅延できる)

## 4. 各画面の改善案

### 4.1 文字起こし画面 (動画レベル設定)

**現状**:
- ☑ 🔒 動画内テキスト自動塗りつぶし

**改善案** (Phase 2 以降):
- 折りたたみ「🎬 動画前処理」セクション化
- 状態表示: cache 済み / 未生成 / 再生成中
- 再生成ボタン

### 4.2 AI自動切り抜きセクション (クリップレベル設定)

**現状不足**: 塗りつぶし版利用チェックボックスなし。

**Phase 1 で対応** (本 PR スコープ):

```
🤖 AI自動切り抜き
  AIが話題を検出し、最適な切り抜きを自動生成します（FCPXML+SRT出力）

  [候補数] [最小秒数] [最大秒数]
  [☑ SRT字幕生成] [☑ 無音削除] [再生速度]
  [ズーム] [アンカーX] [アンカーY] [タイムライン]

  🔒 動画前処理                              ← ★ 新規追加
    ☑ 塗りつぶし版動画を利用 (cache 存在時のみ)

  🎨メディア素材
    ☑ フレーム画像  ☑ BGM  ☑ 効果音  ☑ タイトル画像
    [タイトル幅] [タイトル高さ] [Y位置]
    ☑ 被写体位置からアンカー自動検出

  [🚀 AI自動切り抜きを実行]
```

`export_settings.py` の既存パターン (cache 存在時のみ表示、無ければ ℹ️ メッセージ) を流用。

### 4.3 字幕エディタ画面の DaVinci 送信 (出力レベル設定)

**現状**: チェックボックスを送信ボタン直前に配置済み (PR #136)。

**Phase 2 改善案**:
- 「📺 DaVinci 出力」を expander 化して関連設定を集約
  - ☑ Text+ 自動変換
  - 送信先ビン名 (default: "TextffCut")
  - テンプレート名 (default: "Caption_Default")
  - ☑ 完了後 subtitle track を無効化
- expander 折りたたみ時はサマリ "Text+ ON / TextffCut/Caption_Default" 表示
