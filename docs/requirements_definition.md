TextffCut 要件定義書
1. プロジェクト概要
1.1 製品名
TextffCut（テキストフカット）
1.2 製品概要
YouTube動画のダウンロードから文字起こし、切り抜きを効率化するWebアプリケーション。長時間動画から必要な部分だけを抽出し、無音部分を自動削除してタイトな編集素材を作成することができる。Windows・macOS間の互換性を確保するため、Docker環境での動作を前提とする。
1.3 目的

60分~90分程度の長時間動画の文字起こしと１文字単位の精緻なタイムスタンプを生成
文字起こしテキストを元にdiffライクな直感的なUIで切り抜き部分を指定
タイムライン風のUIで指定箇所のプレビューやカット部分の調整を可能に
無音部分を自動削除しテンポよい動画を自動作成
DaVinci ResolveやFinal Cut Pro等のプロ向け編集ソフトとの連携
字幕ファイルの出力にも対応

1.4 ターゲットユーザー

動画編集者
YouTubeクリエイター
ポッドキャスト制作者
教育コンテンツ制作者
企業の動画マーケティング担当者

1.5 競合分析・差別化要因
主要競合

Adobe Premiere Pro（手動編集中心）
Descript（テキストベース編集だが高価格）
CapCut（モバイル中心、機能制限あり）

差別化要因

精密なタイムスタンプ: 単語レベルの正確な時間軸情報
直感的なテキスト編集: diffライクなUIで視覚的に編集
タイムライン風UI: タイムライン形式でカット部分を直感的に調整
プロツール連携: DaVinci Resolve、Final Cut Pro、Premiere Pro対応
無音自動削除: 手動調整不要のインテリジェント処理

2. 機能要件
2.1 文字起こし機能
2.1.1 ローカル版文字起こし

処理内容: 音声認識による文字起こしと単語レベルのタイムスタンプ生成
処理精度: 中程度の精度モデルを使用（リソースとのバランスを考慮）
対応OS: Docker対応環境（Windows、macOS、Linux）
前処理: 無音部分での自動分割により文字起こし精度を向上
並列処理: ユーザーのPCスペックに応じて最適なワーカー数を自動調整（最大4ワーカー）
自動最適化: 安全性を保ちつつ最大限のPCリソースを活用する動的パラメータ調整
キャッシュ: 処理結果の自動保存と再利用

2.1.2 API版文字起こし

処理方式: クラウドAPIを利用した文字起こし
事前料金確認: 処理前に推定コストを表示
チャンク処理: 無音部分での自然な分割により最大限効果的な処理を実現
後処理: セグメント単位の結果を単語レベルの精密なタイムスタンプに変換

2.1.3 共通機能

対応形式: MP4、MOV、AVI、MKV、WebM等
言語: 日本語、英語等（自動検出）
出力: タイムスタンプ付きテキスト（words単位）
精度: 単語レベルのタイムスタンプ

2.2 テキスト編集機能
2.2.1 編集インターフェース

分割表示: 左側に文字起こし結果の全文表示、右側に切り抜き対象テキストを表示
切り抜き指定: 右側で切り抜きたい部分のテキストを編集し、更新ボタンで左側該当部分をハイライト

2.2.2 編集支援機能

文字数カウント: リアルタイム文字数表示
時間計算: 編集後の推定時間
存在チェック: 削除された文字の自動検出
不一致文字処理: 左側の文字起こしに存在しない文字を一括削除する機能
該当箇所ハイライト: 右側で指定した切り抜き部分に対応する左側文字起こしの箇所を視覚的にハイライト表示
音声プレビュー: 現在切り抜きに指定している箇所を高速再生で音声確認する機能

2.3 タイムライン編集機能
2.3.1 タイムライン表示

セグメント表示: 切り抜きに指定した各部分を個別のセグメントとして並列表示
切り抜き区間表示: 各セグメントの開始/終了点をタイムライン上で表示
波形表示: 音声レベルの可視化によるカット位置の判断支援

2.3.2 微調整機能

数値入力調整: 開始/終了時間を秒単位で直接入力して調整
フレーム単位調整: ±1フレーム、±5フレーム、±30フレームボタンによる段階的調整
キーボードショートカット: 矢印キーによるフレーム単位の微調整操作
プレビュー再生: 調整箇所の前後を再生して確認

2.4 動画処理機能
2.3.1 無音検出・削除

検出内容: 音声レベルに基づく無音部分の自動検出
パラメータ:

閾値: -35dB（調整可能: -60〜-20dB）
最小無音時間: 0.3秒（調整可能: 0.1〜2.0秒）
最小セグメント時間: 0.3秒
パディング: 開始/終了時に0〜2.0秒追加可能



2.3.2 動画切り出し

処理方式: 時間範囲ベースの効率的な切り出し
品質保持: 再エンコードなしの無劣化処理
精度: フレーム単位での正確な切り出し

2.3.3 動画結合

処理内容: 複数の動画セグメントを結合
品質: 無劣化結合
速度: 高速処理（コピーモード）

2.4 エクスポート機能
2.4.1 動画エクスポート

形式: MP4（H.264/AAC）
オプション:

切り抜きのみ
切り抜き＋無音削除



2.4.2 FCPXML（Final Cut Pro XML）

バージョン: 1.9
対応ソフト:

DaVinci Resolve
Final Cut Pro


特徴: アセット参照の最適化

2.4.3 Premiere XML（XMEML）

対応: Adobe Premiere Pro
トラック: ビデオ1＋オーディオ2
パス: 相対パス/絶対パス対応

2.4.4 字幕ファイル

対応形式: SRT
タイムスタンプ: 単語レベルの精密同期
多言語対応: UTF-8エンコーディング

2.5 UI/UX機能
2.5.1 ファイル入力

ファイル選択: 
- Docker環境・ローカル環境共通でvideosフォルダからドロップダウン選択
- Docker環境: /app/videosフォルダ
- ローカル環境: ./videosフォルダ（実行ディレクトリ基準）
- 両環境で同一のUI/UXを提供
検証: ファイル存在確認、形式チェック

2.5.2 進捗表示

プログレスバー: 各処理段階の進捗
ステータス: 詳細な処理状況
キャンセル: いつでも処理中止可能

2.5.3 ダークモード

自動検出: OSの設定に追従
ハイライト調整: 視認性を考慮した色調整

3. 非機能要件
3.1 パフォーマンス要件

文字起こし: 90分動画を15〜30分で処理（ローカル版medium）
無音検出: 90分動画を1〜2分で解析
動画結合: 10セグメントの結合を30秒以内
メモリ使用量: Docker割り当ての80%を上限
レスポンス: UI操作は100ms以内に応答

3.2 キャッシュ・データ保存要件

エラーリカバリー: 処理失敗時の自動リトライ（最大3回）
文字起こし結果の保存: 一度処理した動画の文字起こし結果をJSONファイルとしてアプリケーション展開ディレクトリに保存
処理結果の再利用: 同じ動画ファイルを再度読み込んだ際は保存された文字起こし結果を自動読み込み
設定の永続化: APIキー、無音検出パラメータ、エクスポート設定等のユーザー設定を保存し次回起動時に自動復元

3.3 セキュリティ要件

APIキー: 暗号化保存
ファイルアクセス: Docker環境での隔離
入力検証: ファイル形式、サイズ、内容の検証
ログ: セキュリティイベントの記録

3.4 互換性要件

OS: Windows 10/11、macOS 10.15+、Ubuntu 20.04+（Docker経由）
ブラウザ: Chrome 90+、Firefox 88+、Safari 14+、Edge 90+
Docker: Docker Desktop 4.x以上
メモリ: 最小8GB、推奨16GB以上
ストレージ: 最小20GB空き容量

3.5 保守性要件

単一責任原則: 各モジュールは単一の責任を持つよう設計
依存性注入: 疎結合な設計によるテスタビリティ向上
ログ: 構造化ログによる詳細なエラーログとデバッグ情報
テスト: 単体/統合/E2Eテストの実装（カバレッジ90%以上）
コード品質: Linter、Formatter、型チェックによる品質担保
ドキュメント: README、API仕様書、アーキテクチャ図の整備
バージョン管理: セマンティックバージョニング採用
