# テキスト正規化とタイムスタンプ保持の分析

## 現状の問題点

### 1. 正規化による位置情報の喪失

現在の実装では、テキスト正規化によって文字位置が変化し、タイムスタンプとの対応関係が崩れる可能性があります。

```python
# 例：連続スペースの正規化
元テキスト:     "あいうえお　　かきくけこ" (12文字)
正規化後:       "あいうえお かきくけこ" (11文字)
位置の変化:     位置7-11 → 位置6-10
```

### 2. TimeRangeCalculatorの計算精度

現在の実装：
- セグメント単位での線形補間
- 単語レベルのタイムスタンプは未使用
- 正規化後のテキストで位置を検索し、元のセグメントで時間を計算

### 3. 具体的な問題ケース

1. **連続スペースの圧縮**
   - `re.sub(r'\s+', ' ', text)` により複数の空白が1つに
   - 文字位置がズレる

2. **全角・半角変換**
   - 文字コードは変わるが位置は同じ
   - 検索には影響しない

3. **前後の空白削除**
   - `.strip()` により開始位置がズレる可能性
