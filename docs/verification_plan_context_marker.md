# 文脈マーカー処理改善 - 検証計画

## 作成日: 2025-07-12

## 1. 検証の目的

文脈マーカー処理の新実装が以下の要件を満たすことを確認する：
- 位置計算の正確性
- 「いなこと言うと」のような断片化の解消
- パフォーマンスの維持または向上
- エッジケースへの対応

## 2. 検証環境

### 2.1 テスト環境
- Python 3.11
- 開発環境とDocker環境の両方で実施
- 実際の動画ファイルを使用した統合テスト

### 2.2 テストデータ
- 基本的な日本語テキスト
- 長文テキスト（90分相当）
- 特殊文字を含むテキスト
- 多数の文脈マーカーを含むテキスト

## 3. 検証項目と手順

### 3.1 単体テスト（自動テスト）

#### テストケース1: 基本的な文脈マーカー処理
```python
def test_single_context_marker():
    """単一の文脈マーカーが正しく処理されることを確認"""
    original = "あいうえおかきくけこさしすせそ"
    edited = "あいうえお{かきく}けこさしすせそ"
    
    # 期待結果
    # - "あいうえお" (0-5)
    # - "けこさしすせそ" (8-15)
    # ※ "かきく" (5-8) は除外される
```

#### テストケース2: 複数の文脈マーカー
```python
def test_multiple_context_markers():
    """複数の文脈マーカーが正しく処理されることを確認"""
    original = "あいうえおかきくけこさしすせそ"
    edited = "{あいうえお}かきく{けこ}さしすせそ"
    
    # 期待結果
    # - "かきく" (5-8)
    # - "さしすせそ" (10-15)
```

#### テストケース3: スペースを含むテキスト
```python
def test_with_spaces():
    """スペースが正規化されて正しく処理されることを確認"""
    original = "あいうえおかきくけこさしすせそ"
    edited = "あい うえお{かきく}けこ さしすせそ"
    
    # 期待結果はテストケース1と同じ
```

#### テストケース4: 隣接する文脈マーカー
```python
def test_adjacent_markers():
    """隣接する文脈マーカーが正しく処理されることを確認"""
    original = "あいうえおかきくけこさしすせそ"
    edited = "あいうえお{かきく}{けこ}さしすせそ"
    
    # 期待結果
    # - "あいうえお" (0-5)
    # - "さしすせそ" (10-15)
```

#### テストケース5: 境界調整との組み合わせ
```python
def test_with_boundary_markers():
    """境界調整マーカーと文脈マーカーの組み合わせ"""
    original = "あいうえおかきくけこさしすせそ"
    edited = "あいうえお[<0.1]{かきく}[0.1>]けこさしすせそ"
    
    # 期待結果
    # - "あいうえお" (0-5) ※0.1秒前方に拡張
    # - "けこさしすせそ" (8-15) ※0.1秒後方に拡張
```

## 5. 問題発生時の対応

### 5.1 デバッグ手順
1. ログレベルをDEBUGに設定
2. 位置計算の各ステップを確認
3. 問題のある入力データを最小化
4. ユニットテストで再現

### 5.2 ロールバック計画
- Gitでの変更履歴を保持
- 旧実装への切り替えフラグを用意
- 問題が解決しない場合は旧実装に戻す
