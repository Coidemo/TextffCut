## 3. 実装の詳細

### 3.1 コードの変更箇所

```python
# 主要な変更箇所
1. normalize_with_position_tracking()
   - 位置マッピング配列の追加
   - マーカー位置の自動更新

2. find_differences()
   - ステップ4と5の順序入れ替え（タイムスタンプ取得を早期に）
   - 位置調整計算の簡素化
   - 文脈マーカー除外処理の改善

3. 新規追加メソッド
   - _calculate_position_adjustments()
   - _apply_position_adjustments()
   - _split_range_by_markers()
```

### 3.2 データ構造の変更

```python
# 位置調整情報
position_adjustments = [
    {
        'position': int,        # 調整が適用される位置
        'adjustment': int,      # 累積調整量
        'deleted_length': int   # 削除された長さ
    }
]

# マーカー情報の拡張
marker = {
    'content': str,         # マーカー内のテキスト
    'full_match': str,      # マーカー全体（{}含む）
    'start': int,           # 開始位置
    'end': int,             # 終了位置
    'original_start': int,  # 元の位置（正規化前）
    'original_end': int     # 元の終了位置（正規化前）
}
```

## 4. 実装時の注意点

### 4.1 パフォーマンス
- 位置マッピング配列はテキスト長に比例するため、メモリ使用量に注意
- 大量の文脈マーカーがある場合の処理時間を考慮

### 4.2 互換性
- 既存のインターフェースは変更しない
- 既存のテストケースが全て通過することを確認

### 4.3 エラー処理
- 不正な文脈マーカー形式への対応
- 位置計算のオーバーフロー防止
- 適切なログメッセージの出力
