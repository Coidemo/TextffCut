# タイムライン編集UI改善の要件分析

## 1. 現状の問題点と改善機会

### 1.1 現在の実装状況
- ✅ 基本的なタイムライン編集機能は実装済み
- ✅ セグメントの開始・終了時間を±0.5秒単位で調整可能
- ✅ プレビュー音声再生機能
- ✅ 調整結果の即時反映

### 1.2 ユーザビリティの課題
1. **視覚的フィードバックの不足**
   - 音声波形が表示されないため、無音部分や音声のピークが視覚的に判断できない
   - セグメント間の関係性が分かりにくい

2. **操作性の制限**
   - マウスでのドラッグ操作ができない
   - キーボードショートカットがない
   - 細かい調整（0.1秒単位など）が困難

3. **全体像の把握が困難**
   - タイムライン全体のズーム/パン機能がない
   - 長時間の動画では画面に収まらない

## 2. 改善提案

### 2.1 波形表示機能
**優先度: 高**

#### 要件
- 各セグメントの音声波形を視覚的に表示
- 無音部分と音声部分を明確に区別
- ピークレベルの可視化

#### 技術的考察
- librosaを使用した波形データの取得
- Streamlitのplotly/matplotlibでの描画
- パフォーマンスを考慮したデータの間引き

### 2.2 インタラクティブな操作性
**優先度: 中**

#### 要件
- セグメント境界のドラッグ操作
- ダブルクリックでのプレビュー再生
- 右クリックメニューでの操作

#### 技術的考察
- Streamlitの制限内での実装
- カスタムコンポーネントの必要性検討
- JavaScriptとの連携

### 2.3 キーボードショートカット
**優先度: 中**

#### 要件
- Space: 再生/停止
- ←/→: 前後のセグメントへ移動
- Shift + ←/→: 境界を0.1秒単位で調整
- Ctrl + Z/Y: 元に戻す/やり直し

#### 技術的考察
- Streamlitのキーイベント処理
- ブラウザのデフォルト動作との競合回避

### 2.4 ズーム・パン機能
**優先度: 中**

#### 要件
- タイムライン全体の表示倍率変更
- 水平スクロール
- フォーカスしたい部分への自動スクロール

#### 技術的考察
- CSSでのオーバーフロー処理
- ズームレベルの保持と同期

### 2.5 詳細編集モード
**優先度: 低**

#### 要件
- 0.01秒単位での微調整
- フレーム単位での編集
- 数値入力による正確な時間指定

#### 技術的考察
- 既存の±0.5秒調整との共存
- UIの複雑化を避ける設計

## 3. 実装優先順位

### Phase 1: 視覚的改善（1-2日）
1. **波形表示の基本実装**
   - 音声データの取得と処理
   - シンプルな波形描画
   - セグメント単位での表示

2. **UIレイアウトの改善**
   - 波形表示エリアの追加
   - 既存コントロールの再配置
   - レスポンシブデザイン

### Phase 2: インタラクション改善（2-3日）
1. **基本的なマウス操作**
   - セグメント選択
   - 境界調整（制限付き）

2. **キーボードショートカット**
   - 基本的なナビゲーション
   - 再生制御

### Phase 3: 高度な機能（3-4日）
1. **ズーム・パン機能**
   - 表示倍率の変更
   - スクロール同期

2. **詳細編集モード**
   - 高精度調整
   - 数値入力

## 4. 技術的制約と対策

### 4.1 Streamlitの制限
- **問題**: ネイティブアプリのような高度なインタラクションが困難
- **対策**: 
  - 段階的な実装で基本機能を優先
  - 必要に応じてカスタムコンポーネント開発を検討

### 4.2 パフォーマンス
- **問題**: 大量の波形データ処理による遅延
- **対策**:
  - データの事前処理とキャッシング
  - 表示解像度の動的調整
  - 非同期処理の活用

### 4.3 ブラウザ互換性
- **問題**: 異なるブラウザでの動作差異
- **対策**:
  - 主要ブラウザでのテスト
  - フォールバック機能の実装

## 5. 成功指標

1. **ユーザビリティ向上**
   - セグメント境界調整の所要時間50%削減
   - 誤操作によるやり直し回数の減少

2. **視覚的理解の向上**
   - 無音部分の識別精度向上
   - 音声ピークに基づく適切な分割

3. **操作効率の改善**
   - キーボードのみでの基本操作完結
   - マウスドラッグによる直感的な調整
