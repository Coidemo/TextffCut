# タイムライン編集UI改善 - 基本設計書 v2

## 1. 設計概要

### 1.1 目的
タイムライン編集機能のユーザビリティを抜本的に向上させ、**「クリップベースのビジュアル編集」**による、直感的で高速かつ正確な操作体験を実現する。

### 1.2 設計方針
- **UXの抜本的改善**: 従来のUIを刷新し、音声波形を直接操作するUIを導入する。
- **カスタムコンポーネントの積極活用**: 高度なインタラクティブ性を実現するため、Streamlitカスタムコンポーネントを開発・導入する。
- **パフォーマンス優先**: 長時間の動画から抽出された多数のクリップでも、快適に動作する設計とする。
- **段階的実装**: まずはコアとなる波形表示と境界調整機能を実装し、その後、分割やスナップなどの高度な機能を追加する。

## 2. 機能設計

### 2.1 Phase 1: クリップベース・タイムラインの基盤構築

#### 2.1.1 概要
ユーザーが切り抜き対象として指定したテキスト部分に対応する**動画クリップ群**を、連結された一本の仮想タイムラインとして表示する。各クリップは**実際の音声波形**と共に描画され、ユーザーは最終的な出力動画の構成を視覚的に把握できる。

#### 2.1.2 UI/UXデザイン（ワイヤーフレーム）
```
【タイムライン編集エリア】

[動画プレビュー（現在カーソル位置の映像を表示）]

┌──────────────────────────────────────────────────────────────────────────┐
│ [クリップ1: 0:10-0:18]  [クリップ2: 0:45-0:52]  [クリップ3: 1:12-1:20] ... │
│<|>    ╱╲╱╲            <|>    ╱╲╱╲            <|>    ╱╲╱╲              <|>│
│ │    ╱  ╲  ╲...         │    ╱  ╲  ╲...         │    ╱  ╲  ╲...           │
│<|>   ╱    ╲╱             <|>   ╱    ╲╱             <|>   ╱    ╲╱             <|>│
│ └───────────┘         └───────────┘         └───────────┘            │
│ ▲(再生ヘッド)                                                            │
└──────────────────────────────────────────────────────────────────────────┘
 [ズームスライダー] [-][+]  [再生/停止ボタン] [現在時間] / [総時間]
```
- **クリップ**: 切り抜き対象の各部分。タイムライン上では連結して表示。
- **波形**: `librosa`で抽出した実際の音声波形。
- **境界ハンドル (`<|>`):** 各クリップの開始点と終了点。マウスでドラッグして時間を調整可能。
- **再生ヘッド (`▲`):** 現在の再生位置。キーボードの矢印キーやクリックで移動可能。
- **動画プレビュー**: 再生ヘッドの位置に対応する**元の動画**のフレームを表示し、編集コンテキストを明確にする。

#### 2.1.3 データフロー
1.  **Python (Backend)**:
    -   `TextEditingService`が、編集テキストから時間範囲のリスト（クリップ群）を特定する。
    -   `WaveformProcessor`が、各クリップの音声波形データを動画ファイルから抽出し、ダウンサンプリングしてJSON化する。
    -   `main.py`が、クリップの時間情報と波形データをカスタムコンポーネントに渡す。
2.  **JavaScript (Frontend - カスタムコンポーネント)**:
    -   `wavesurfer.js`等のライブラリを使用し、受け取ったデータからタイムラインと波形を描画する。
    -   ユーザーのドラッグ操作やクリックを検知し、UIを更新する。
    -   編集結果（変更後の時間範囲リスト）をPython側に送信する。
3.  **Python (Backend)**:
    -   カスタムコンポーネントから返された時間範囲を受け取り、`st.session_state`に保存する。

#### 2.1.4 データ構造
```python
# Python側からコンポーネントに渡すデータ
class ClipData:
    id: str  # "clip-0", "clip-1", ...
    start_time: float # 元動画における開始時間
    end_time: float   # 元動画における終了時間
    waveform: list[float] # 正規化された波形サンプル

# コンポーネントからPython側に返すデータ
class AdjustedTimeRanges:
    ranges: list[tuple[float, float]]
```

### 2.2 Phase 2: 高度な編集機能

#### 2.2.1 クリップの分割・結合
- **分割**: 再生ヘッドの位置でクリップを2つに分割する機能（`S`キーなど）。
- **結合**: 隣接するクリップを1つに結合する機能。

#### 2.2.2 スナップ機能
- クリップの境界をドラッグする際、他のクリップの境界や文字起こしの単語境界に吸着する機能。

### 2.3 Phase 3: キーボードショートカットの拡充
| キー | 動作 |
|:---|:---|
| `Space` | 再生/一時停止 |
| `←` / `→` | 再生ヘッドを0.1秒移動 |
| `Shift` + `←` / `→` | 再生ヘッドを1秒移動 |
| `I` / `O` | 再生ヘッド位置にIn点/Out点を設定 |
| `S` | 再生ヘッド位置でクリップを分割 |
| `Ctrl/Cmd` + `Z` | 元に戻す |
| `Ctrl/Cmd` + `Shift` + `Z` | やり直し |

## 3. 技術仕様

### 3.1 依存ライブラリ
- **Python**: `librosa>=0.10.0`
- **JavaScript**: `streamlit-component-lib`, `wavesurfer.js` (または類似の波形描画ライブラリ)

### 3.2 `WaveformProcessor`の改修
`extract_waveform_for_clips(video_path: str, time_ranges: list[tuple[float, float]])` メソッドを新設。指定された複数の時間範囲の波形を一括で抽出し、クリップごとの`WaveformData`リストを返す。

### 3.3 カスタムコンポーネントの責務
- **入力**: クリップデータ（時間、波形）、UI設定（色など）
- **処理**: タイムライン描画、ユーザーインタラクションの処理
- **出力**: ユーザーによって変更された後の時間範囲リスト

## 5. リスクと対策

| リスク | 対策 |
|:---|:---|
| **カスタムコンポーネント開発の複雑性** | `wavesurfer.js`など実績のあるライブラリを活用。まずは最小限の機能（表示とドラッグ）に絞って実装し、段階的に拡張する。 |
| **多数のクリップによるパフォーマンス低下** | 仮想レンダリング（画面に見えている範囲だけを描画する）技術をフロントエンドで採用する。 |
| **ブラウザ間の挙動差異** | 主要ブラウザ（Chrome, Firefox, Safari）での動作確認を徹底する。 |
