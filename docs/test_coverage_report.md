## テスト戦略

### 1. 単体テストの方針
- 各クラス・メソッドを独立してテスト
- 外部依存はモックで置き換え
- 境界値と異常系を網羅
- 不変性とスレッドセーフティを確認

### 2. 統合テストの方針
- 実際のDIコンテナを使用
- 層間の連携を確認
- エラーが適切に伝播することを確認
- パフォーマンスへの影響を最小限に

### 3. テストの保守性
- Fixtureを活用して共通処理を集約
- テストケースは独立して実行可能
- 明確な命名規則でテストの意図を表現
- アサーションは具体的で検証可能
