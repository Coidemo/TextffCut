# タイムライン編集UI再設計

## 現在の問題点
1. セグメントごとに個別に表示され、全体像が把握しづらい
2. 「編集を完了」ボタンが機能しない（Streamlit内でのreturn問題）
3. 操作が直感的でない
4. **境界調整結果が波形表示に反映されない**
5. **後方のセグメントで波形とスライダーが離れていて操作しづらい**
6. **スライダーでは細かい編集ができない（0.1秒単位が限界）**

#### 5.2 実装方針

1. **リアルタイム波形更新**
   - 境界調整時に波形表示も即座に更新
   - `st.rerun()`を使わず、Plotlyの`update_traces`を活用

2. **セグメント選択の改善**
   - 波形上の任意の位置をクリックで該当セグメントを選択
   - 選択セグメントは下部の拡大エリアに表示

3. **精密編集コントロール**
   ```python
   # 数値入力（ミリ秒単位まで対応）
   col1, col2 = st.columns(2)
   with col1:
       time_str = st.text_input(
           "開始時間", 
           value=format_time_with_ms(segment.start),
           placeholder="00:00:00.000"
       )
   
   # ボタングループ（フレーム単位の調整含む）
   adjustment_buttons = st.columns(6)
   adjustments = [
       ("-1s", -1.0),
       ("-0.1s", -0.1),
       ("-1f", -1/fps),  # フレーム単位
       ("+1f", 1/fps),
       ("+0.1s", 0.1),
       ("+1s", 1.0)
   ]
   ```

4. **波形とコントロールの近接配置**
   - 選択したセグメントの拡大波形を編集コントロールの直上に配置
   - スクロール不要な設計
