# 詳細設計仕様書v3 - テキスト処理関連の更新

## 更新日: 2025-07-12

## 1. 更新の背景

文脈マーカー `{}` の処理において位置計算の複雑さによるバグが発生したため、処理フローを見直し、より単純で確実な実装に変更します。

## 2. 詳細設計仕様書v3への追加項目

### 2.1 モジュール間依存関係への影響

| 変更対象モジュール | 影響を受けるモジュール | 影響内容 |
|-------------------|---------------------|----------|
| adapters.gateways.text_processing.sequence_matcher_gateway | services.text_editing_service | 文脈マーカー処理の内部実装変更（インターフェースは維持） |
| adapters.gateways.text_processing.sequence_matcher_gateway | domain.use_cases.text_difference_detector | タイムスタンプ取得タイミングの変更 |

### 2.2 テキスト処理の詳細設計

#### 2.2.1 文脈マーカー処理の新フロー

**処理の流れ:**
1. 文脈マーカーの位置を記録（元のテキストの位置）
2. テキストを正規化（スペース削除）しながら位置を追跡
3. 文脈マーカーを一時的に削除（差分検出のため）
4. SequenceMatcherで差分検出
5. 差分検出結果から即座にタイムスタンプを取得
6. 文脈マーカー位置を復元して、その部分を除外
7. 境界調整を適用

**主要メソッドの変更:**

```python
# SequenceMatcherTextProcessorGateway.find_differences()の変更点

# 旧実装の問題点:
# - 文脈マーカー削除後に複雑な位置復元を行っていた
# - 位置計算の誤りにより断片化が発生

# 新実装の改善点:
# - 位置追跡を明確に分離
# - タイムスタンプを差分検出直後に取得
# - 位置調整の計算を簡素化
```

#### 2.2.2 位置追跡の実装詳細

```python
def normalize_with_position_tracking(self, text: str, markers: List[dict]) -> Tuple[str, List[dict], List[int]]:
    """
    テキストを正規化しながら位置を追跡

        - 更新されたマーカー（正規化後の位置）
        - 位置マッピング（元の位置→正規化後の位置）
    """
```

#### 2.2.3 エラーハンドリングの追加

- 文脈マーカーが重複する場合の処理
- 不正な文脈マーカー形式の検出
- 位置計算のオーバーフロー防止

### 2.3 テストケースの追加

|---------------|-----------|------|

### 2.4 パフォーマンスへの影響

- **改善点**: タイムスタンプ取得を早期に行うことで、後続の位置復元処理を削減
- **メモリ使用量**: 位置マッピング配列の追加により若干増加（元テキストの文字数分）
- **処理時間**: 全体的に高速化（複雑な位置計算の削除により）

## 3. 実装時の注意事項

### 3.1 後方互換性の維持

- 外部インターフェース（`ITextProcessorGateway`）は変更なし
- 既存のテストケースは全て通過すること

### 3.2 デバッグ情報の追加

- 位置計算の各ステップでログ出力
- 文脈マーカーの処理状況を可視化

### 3.3 エッジケースの考慮

- 空の文脈マーカー `{}`
- ネストした文脈マーカー（非対応だがエラーメッセージを表示）
- 極端に長い文脈マーカー
