## 最重要ルール
- タイトルの文字は一切変更・省略・言い換えしないこと
- 各デザインの全セグメントのtextを結合した結果が元タイトルと完全一致すること

## デザインバリエーション指示
{NUM_CANDIDATES}パターンそれぞれで以下を変えてください:

### フォントサイズ戦略
- パターン1-2: 大胆（font_size 160-200、少ない行数で大きく）
- パターン3-4: 標準（font_size 120-160、バランス重視）
- パターン5-6: コンパクト（font_size 90-130、3行で確実に収まる）

### 配色バリエーション（必ず異なる配色にすること）
- 暖色系: 赤・オレンジ・黄のグラデーション（例: #FF4444→#FF8800）
- 寒色系: 青・水色・紫のグラデーション（例: #4488FF→#00CCFF）
- ビビッド: 鮮やかなコントラスト色（例: #FF00FF→#FFFF00）
- モノトーン+アクセント: 白文字に一箇所だけ鮮やかな色
- ゴールド系: 金色グラデーション（例: #FFD700→#FF8C00）
- ネオン系: 蛍光色グラデーション（例: #00FF88→#00FFFF）

### 行分割パターン
- 2行分割と3行分割を混ぜる
- 意味の切れ目を変えて異なる印象を作る

## デザインルール（全パターン共通）
1. 2-3行に分割（意味の切れ目、インパクト重視で改行）
2. 各行内をさらにセグメントに分割（強調語・句読点・助詞などで区切る）
3. 背景フレームの色に映える配色を選ぶ
4. 白文字や明るい色の文字は黒の外アウトラインのみ（inner_outline_width=0）でシンプルに
5. グラデーションセグメントのみ二重アウトライン（外側=暗色、内側=明色）で装飾する
6. 強調セグメントにgradientを使う（2色の縦グラデーション）
7. weightは強調語=Eb、補足=Bd、句読点=Rg

## 出力JSON
以下のスキーマに従ってJSON**のみ**を出力してください:
```json
{
  "designs": [
    {JSON_SCHEMA},
    ...
  ]
}
```

### フィールド説明
- text: セグメントの文字列
- font_size: 80-220px
- color: 単色の場合のhex色コード（gradient指定時は無視される）
- gradient: 縦グラデーション ["開始色", "終了色"] またはnull
- weight: フォントウェイト Th/Rg/Bd/Eb
- outer_outline_color: 外側アウトライン色（行全体共通）
- outer_outline_width: 外側アウトライン太さ（0-10）
- inner_outline_color: 内側アウトライン色
- inner_outline_width: 内側アウトライン太さ（0-10、0で無効）
- line_spacing: 行間px（0-50）
- padding_top: 上端からのpx（0-200）
