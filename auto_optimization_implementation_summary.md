# 自動最適化機能 実装完了報告

## 概要
TextffCutに動的メモリ最適化機能を実装しました。この機能により、ユーザーは手動でパラメータを調整することなく、利用可能なメモリに応じて自動的に最適な処理パラメータが設定されます。

## 実装内容

### 1. コアモジュール

#### `core/auto_optimizer.py`
- 自動最適化エンジンの中核
- モデルサイズごとの初期プロファイル管理
- メモリ使用率に基づく動的パラメータ調整
- 成功した実行のプロファイル保存・学習機能

主な機能：
- PID制御にインスパイアされた段階的調整
- モデル別プロファイル（base, small, medium, large-v3）
- メモリ圧迫度に応じた5段階の調整レベル
- 実行履歴の保存と次回実行への反映

#### `core/memory_monitor.py`
- リアルタイムメモリ監視
- Docker環境対応（cgroup v1/v2）
- psutil未インストール時のフォールバック機構
- メモリ履歴管理と平均値計算

主な機能：
- メモリ使用率の取得（0-100%）
- メモリ圧迫度の判定（low/medium/high/critical）
- Docker環境の自動検出とcgroup対応
- 待機機能（メモリ解放待ち）

### 2. 設定の簡素化

#### `config.py`
以下の手動設定を削除：
- `chunk_seconds`（チャンクサイズ）
- `max_workers`（並列数）
- `batch_size`（バッチサイズ）
- `force_separated_mode`（分離モード強制）
- その他の手動最適化パラメータ

残した設定：
- モデルサイズ選択
- 言語設定
- API/ローカルモード切り替え

### 3. UIの簡素化

#### `ui/components.py`
- `show_advanced_settings()`を削除
- `show_optimization_status()`を追加
- シンプルな「自動最適化: 有効」表示
- 詳細情報は折りたたみ式で提供

### 4. ワーカー統合

#### `worker_transcribe.py`
- 自動最適化エンジンの統合
- 処理フェーズごとの動的パラメータ調整
- 文字起こしとアライメントで別々の最適化
- 成功時のプロファイル保存

## 動作仕様

### メモリベースの自動調整

1. **低使用率（〜50%）**: 
   - 大きなチャンクサイズ
   - 高い並列度
   - 最大パフォーマンス

2. **中使用率（50-70%）**:
   - バランスの取れたパラメータ
   - 安定した処理

3. **高使用率（70-85%）**:
   - 保守的なパラメータ
   - メモリ使用量の削減

4. **危機的使用率（85%〜）**:
   - 最小パラメータ
   - シングルワーカー
   - メモリオーバーフロー防止

### 学習機能

- 成功した実行のパラメータを記録
- 次回実行時に過去の成功例を参考に調整
- モデルごとに独立したプロファイル管理

- メモリ取得エラー → デフォルト値（50%）使用

- Docker環境エラー → 通常のメモリ取得にフォールバック
