## 🎯 基本的な使い方

### 1. アプリケーションの起動

```bash
# ローカル版
./run.sh

# または
source venv/bin/activate
streamlit run main.py

# Docker版
./docker-run.sh start
```

ブラウザが自動的に開き、TextffCutが表示されます。

### 2. 基本的なワークフロー

1. **動画ファイルの選択**
2. **文字起こし実行**
3. **テキスト編集で切り抜き箇所を選択**
4. **無音削除の設定（オプション）**
5. **出力形式を選択して実行**

## 📝 文字起こし機能

### モード選択

#### ローカルモード（デフォルト）
- **特徴**: PC上で処理、無料、プライベート
- **推奨**: GPU搭載Mac、高性能PC

#### APIモード
- **特徴**: クラウド処理、高速、有料（$0.006/分）
- **推奨**: GPU非搭載、処理速度重視

### Whisperモデルの選択

| モデル | 精度 | 速度 | 推奨用途 |
|--------|------|------|----------|
| base | 低 | 最速 | テスト・下書き |
| small | 中 | 速い | 短い動画 |
| medium | 高 | 普通 | 一般的な用途 |
| large-v3 | 最高 | 遅い | 高精度が必要な場合 |

### 文字起こしのコツ

- **長時間動画**: チャンク分割で並列処理されるため効率的
- **音質改善**: 事前にノイズ除去すると精度向上
- **キャッシュ活用**: 同じ動画は2回目以降高速

## ✂️ 切り抜き編集

### 基本的な編集

1. **残したい部分をコピー**
   - 文字起こし結果から必要な部分を選択
   - Cmd+C（Mac）でコピー

2. **編集エリアにペースト**
   - 編集用テキストエリアにペースト
   - 複数箇所をペーストして結合可能

### 高度な編集テクニック

#### 複数セクション機能

区切り文字（`---`）を使って複数の独立したセクションを作成：

```
セクション1の内容...

---

セクション2の内容...

---

セクション3の内容...
```

各セクションが独立した動画として処理されます。

#### タイムスタンプの活用

文字起こし結果には時間情報が含まれています：
- 「[00:01:23]」のような表記で位置を確認
- より正確な編集が可能

## 🔇 無音削除機能

### 基本設定

- **閾値**: -35dB（デフォルト）
  - 小さくする：より多くの音を無音とみなす
  - 大きくする：はっきりした無音のみ削除

- **最小無音時間**: 0.3秒
  - 短い無音は削除されない

- **PAD設定**: 0.1秒（デフォルト）
  - セグメント前後の余白
  - 自然なつなぎのために重要

### 推奨設定

| コンテンツ種別 | 閾値 | 最小無音時間 | PAD |
|---------------|------|-------------|-----|
| インタビュー | -35dB | 0.5秒 | 0.2秒 |
| プレゼン | -40dB | 0.3秒 | 0.1秒 |
| 音楽含む | -30dB | 1.0秒 | 0.3秒 |

## 📤 出力オプション

### 1. 動画ファイル（MP4）

- **用途**: すぐに使える完成品
- **特徴**: 元の品質を維持
- **出力先**: `{動画名}_TextffCut/`フォルダ

### 2. FCPXML

- **用途**: Final Cut Pro / DaVinci Resolve での編集
- **特徴**: タイムライン情報を保持
- **メリット**: 後から微調整可能

### 出力ファイルの命名規則

```
元動画名_TextffCut/
├── 元動画名_cut.mp4          # 切り抜き動画
├── 元動画名_no_silence.mp4   # 無音削除版
├── 元動画名.fcpxml           # 編集データ
└── transcriptions/           # 文字起こしキャッシュ
```

## 🚀 高度な機能

### 処理モードの組み合わせ

1. **切り抜きのみ**: 高速、正確
2. **切り抜き＋無音削除**: タイトな編集
3. **FCPXMLエクスポート**: 後編集用

### パフォーマンス最適化

- **並列処理**: 自動的に最適化
- **メモリ管理**: 大きなファイルも効率的に処理
- **プログレスバー**: リアルタイムで進捗確認

### ショートカット

- **Cmd+Enter**: 処理開始（テキストエリア内）
- **Esc**: 処理中止

## 💡 ベストプラクティス

### 効率的なワークフロー

1. **事前準備**
   - 動画の音声品質を確認
   - 不要な部分を大まかに把握

2. **文字起こし**
   - 初回は`medium`モデルでテスト
   - 本番は`large-v3`で高精度化

3. **編集**
   - まず大まかに切り抜き
   - プレビューで確認後、微調整

4. **出力**
   - テスト出力で確認
   - 問題なければ本番出力

### よくあるミス

- ❌ 存在しないテキストを追加
- ❌ タイムスタンプを編集
- ❌ 改行を過度に変更

### 推奨事項

- ✅ オリジナルの文言を維持
- ✅ 自然な区切りで編集
- ✅ 定期的に保存（ブラウザ機能）

## 🎓 実践例

### YouTubeコンテンツの編集

1. 90分のライブ配信から10分のハイライト作成
2. 「えー」「あのー」を含む部分を削除
3. 無音削除でテンポアップ
4. 結果：10分の濃縮版完成

### インタビュー動画の整理

1. 60分のインタビューから重要部分抽出
2. 質問ごとにセクション分割（`---`）
3. 各セクションを個別に出力
4. 結果：トピック別の短編動画

## 🔍 トラブルシューティング

### よくある質問

**Q: 文字起こしが途中で止まる**
A: メモリ不足の可能性。より小さいモデルを使用するか、動画を分割

**Q: 無音削除後の音が不自然**
A: PAD設定を増やす（0.2〜0.3秒）

**Q: 出力ファイルが見つからない**
A: 動画と同じフォルダの`_TextffCut`サブフォルダを確認

**Q: APIキーエラー**
A: キーの先頭に空白がないか、`sk-`で始まっているか確認
